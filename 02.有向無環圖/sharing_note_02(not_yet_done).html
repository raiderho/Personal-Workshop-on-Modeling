1. 有向無環圖 (Directed Acyclic Graph, DAG)

  • 這次分享基本上取自 Scott Cunningham - Causal Inference_ The Mixtape (2021) 第 3 章

  • 圖形因果建模的歷史可以追溯到 20 世紀初的 Sewall Wright, 他是現代遺傳學之父之一，也是經濟學家 Philip Wright 的兒子。

  • Sewall 開發了遺傳學路徑圖，而 Philip 則將其應用於計量經濟學識別。然而，經濟學界（與統計學界） 基本上無視了圖形建模在因果推理中的應用。

  • 透過電腦科學家、圖靈獎得主 Judea Pearl 的努力，圖形推理在因果推理方面才復興。

    • 他在 Judea Pearl - Causality (2009, 2nd ed.) 中解釋了這一點，本書是因果推理的一般理論，闡述了有向圖的用途。

  • 圖形模型對設計可靠的識別 (identification) 策略大有幫助。

1-1. 補充: 何謂「識別」？

  • 在統計學和計量經濟學中，識別指的是模型的結構參數能夠被唯一且明確地估計出來的條件。

  • 具體而言，識別意味著觀察到的數據提供了足夠的資訊，從而使模型中的未知參數能夠被唯一確定。

  • 識別問題的三種類型:

    • 完全識別 (Exactly Identified):
      當數據中提供的資訊（觀測方程的數量）等於模型中未知參數的數量時，模型是完全識別的。此時參數可以唯一確定。

    • 過度識別 (Over-Identified):
      當數據提供的資訊多於未知參數的數量時，模型是過度識別的。這通常被視為有利的情況，因為我們可以檢驗模型的適配度，並確保參數估計的一致性。

    • 未識別 (Under-Identified):
      當數據提供的資訊少於模型中未知參數的數量時，模型是未識別的。此時，無法唯一確定參數，因為不同的參數組合可能會導致相同的觀察結果。

  • 識別的重要性:
    識別是統計建模中的一個核心問題，因為如果模型未識別，無法確定我們所估計的參數是否有意義或能夠反映真實的現象。

1-2. 符號說明

  • 我們將 DAG 視為因果效應鏈的圖形表示:
    因果效應本身基於一些潛在的、無法觀察到的結構化過程，所有這些都可以用節點和箭頭等圖形符號表示出來。

    • 節點代表隨機變數，而這些隨機變數假定是由某個資料生成過程 (data generating process) 產生的。

    • 箭頭代表兩個隨機變數之間的因果效應，箭頭的方向反映了因果關係的方向。

  • 在 DAG 中，因果關係是單向的。具體而言，它在時間上是向前的。DAG 中沒有迴圈 (cycle)。

    • 要顯示反向因果關係，需要創建多個節點，很可能是同一節點的兩個版本，以時間索引 (time index) 區隔。

    • 同樣，供需模型中的同時性也不能直接用 DAG 來表示，建議採用不同的方法。

  • DAG 用反事實來解釋因果關係，因果效應被定義為世界兩種狀態之間的比較:
    一種狀態是當某種干預取某些值（實際發生的值），另一種狀態是在干預取其他值（反事實）。

  • 因果效應有兩種方式:

    • 直接影響，例如: \(D \to Y\)

    • 由第三個變數中介 (mediate): \(D \to X \to Y\)

  • DAG 意在描述與 \(D\) 對 \(Y\) 的影響相關的所有因果關係:
    既明確描述了因果效應路徑，而缺少箭頭表示缺少因果路徑。 因此，DAG 既包含連接變數的箭頭，也包含排除哪些箭頭。

  • 一個完整的 DAG 將包含圖中變數之間的所有直接因果效應，以及圖中任何一對變數的所有共同原因。

1-3. 補充: DAG 從何而來？

  • 經驗、信念、探索性分析、因果發現…

  • DAG 是最新知識的理論表示。這些知識來自於各種不同的來源。例如：科學模型、與專家的對話、自己的觀察和經驗、文獻綜述，以及您自己的直覺和假設。

1-4. DAG 的積極意義

  • DAG 對於理解先驗知識在確定因果效應中的關鍵作用非常有用。

  • DAG 對於交流研究設計和估計方法非常有幫助。

    • 工具變數 (instrument variable) 尤其如此，因為工具變數有非常直觀的 DAG 表示。

    • 透過後門準則 (backdoor criterion) 和衝突 (colliding) 等概念，設計良好的 DAG 可以協助發展出可靠的研究設計，以確定某些干預措施的因果效應。

  • DAG 讓我們明白了一個道理：所有因果效應的識別都需要假設。

2. 線性迴歸的假設與工具變量

  • 線性迴歸的假設是非常重要的，主要是誤差項 \(u\) 與解釋變數 \(x\) 必須無關: \[y= \beta_0 + \beta_1 x + u, \quad \text{E}[u]=0, \; \color{red}{\text{Cov}(x,u)=0}.\]

  • 若此假設違反，常見的統計估計方法（最小平方法）估計出的係數不可信

    • 通常說的是違反一致性，即不一致 (inconsistent): \(\hat{\beta_1} \overset{p}{\to} \beta_1\) 不再成立。

  • 經濟學對此議題發展出一整套處理手法，稱為工具變量 (instrumental variable, instrument)，簡稱 IV

  • 推薦簡錦漢對 IV 的介紹

3. DAG 的正式介紹

3-1. 簡單的 DAG

3-1-1. 可觀測的干擾因子 \(X\)

  • 在底下的 DAG 中我們有三個隨機變數: \(D, X, Y\). 考慮所有連接 \(D\) 與 \(Y\) 的可能路徑:

  • 第一條直接路徑從 \(D\) 到 \(Y\):
    \(D \to Y\)，這是因果路徑。

  • 第二條路徑連接 \(D\) 與 \(Y\):
    \(D \leftarrow X \to Y\)， 稱為後門路徑 (backdoor pathway)。後門路徑不是 \(D\) 與 \(Y\) 之間的因果關係。

  • 後門路徑的想法是我們可以從 DAG 中學到的最重要的內容。

    • 它等同於遺漏變數偏差，因為 \(X\) 同時影響結果（此指 \(Y\)）和處理變數（treatment variable，此指 \(D\)）。

  • 在迴歸中不控制變數會導致遺漏變數偏差一樣，讓後門打開也會產生偏差。

    • 因此，我們當然想要關閉後門！

  • 我們稱 \(X\) 為干擾因子 (confounder)，因此它干擾了我們評估 \(D\) 對 \(Y\) 效應的能力。

3-1-2. 無法觀測的干擾因子 \(U\)

  • 在底下的 DAG 中我們有三個隨機變數: \(D, U, Y\).

  • 在前面的例子中，\(X\) 可以被觀察到，它指向 \(D\) 與 \(Y\) 的箭頭都是直線。

  • 在這個例子，干擾因素 \(U\) 無法被觀測到，我們用虛線表示 \(U\) 指向 \(D\) 與 \(U\) 指向 \(Y\)。

  • 性質: 若後門路徑中的干擾因子無法被觀察，（若沒有被控制或處理，）後門路徑是打開的。

3-1-3. 衝突因子與後門準則

  • 在底下的 DAG 中我們有三個隨機變數: \(D, X, Y\).

  • 第一條 \(D \to Y\)，這是因果路徑。

  • 第二條路徑連接 \(D\) 與 \(Y\):
    \(D \to X \leftarrow Y\) 是後門路徑。

    • 現在 \(X\) 有兩個箭頭指向它，而非遠離它。

  • 在這個例子，\(X\) 稱為衝突因子 (collider)。

  • 後門準則:

    • 沒有去控制衝突因子時，衝突因子會自動關閉相應的後門路徑。
    • 控制衝突因子時，反而會打開（原本已經關閉的）後門路徑。
    • 對於混淆因子造成的後門，若能控制（條件化）後門路徑上的非衝突因子，可以關閉這條後門路徑。

3-2. 應用例子

3-2-1. 教育的影響

  • \(Y\): 關心對象的收入; \(D\): 大學教育; \(PE\): 父母的教育程度; \(I\): 家庭收入; \(B\): 觀測不到的背景

  • 我們關心 \(D\) 對 \(Y\) 的影響

  • 連接 \(D\) 到 \(Y\) 的路徑:

    • 直接因果路徑: \(D \to Y\)

    • 後門路徑1: \(D \leftarrow I \to Y\)

    • 後門路徑2: \(D \leftarrow PE \to I \to Y\)

    • 後門路徑3: \(D \leftarrow B \to PE \to I \to Y\)

  • 因果推斷 (causal inference) 的策略:
    \(I\) 不是衝突因子，所有的後門路徑都通過 \(I\)，所以控制 \(I\) 就可以關閉所有的後門。

    • 比如: 在迴歸分析中，考慮 \(Y=\beta_0 + \beta_1 D + \beta_2 I + \varepsilon\).

  • 現在，考慮另一種可能情況:

  • \(D\) 到 \(Y\) 之間多了（至少）一條後門路徑: \(D \leftarrow B \to Y\)

  • 此時，控制 \(I\) 是不夠的，這一條後門路徑的影響沒有被消除，估計 \(D \to Y\) 的因果效應會有偏誤。

  • 不同的假設，就會決定如何估計 \(D\) 對 \(Y\) 的因果效應。

3-2-2. 稍微複雜的模型

  • 連接 \(D\) 到 \(Y\) 的後門路徑:

    • 後門路徑1: \(D \leftarrow U_1 \to Y\)

    • 後門路徑2: \(D \leftarrow U_2 \to Y\)

    • 後門路徑3: \(D \leftarrow U_1 \to I \leftarrow U_2 \to Y\)

    • 後門路徑4: \(D \leftarrow U_2 \to I \leftarrow U_1 \to Y\)

  • 此時，控制 \(I\) 無濟於事，因為無法關閉後門路徑1與後門路徑2。

  • 事實上，控制 \(I\) 還更糟，因為會打開後門路徑3與後門路徑4。

3-2-3. 性別歧視的模型

  • \(F\): 女性; \(D\): 歧視; \(O\): 職業; \(Y\): 收入; \(A\): 能力

  • 連接 \(D\) 到 \(Y\) 的路徑:

    • 直接因果路徑: \(D \to Y\)

    • 間接因果路徑: \(D \to O \to Y\)

    • 後門路徑: \(D \to O \leftarrow A \to Y\)

  • 控制 \(O\) 會發生什麼事情呢？
    因為 \(O\) 是衝突因子，控制 \(O\) 反而會打開後門路徑，對 \(D \to Y\) 的因果效應的估計產生偏誤。

  • 底下的模擬將顯示這一點。

    • 補充1: 在這個模型中（或者說從圖來看），\(F\) 是 \(D\) 的工具變數。不過，在底下的模擬中，簡化成 \(F\) 就是 \(D\)。

    • 補充2: 在圖中，能力 (ability) 是不可觀測的變量，但在模擬中，在第3條回歸式我們直接將 \(Y\)（即 wage）對 \(F\)（即 female）、\(O\)（即 occupation）與 \(A\)（即 ability）作迴歸，這只是為了方便展示。

library(tidyverse)
library(stargazer)
tb <- tibble(
  female = ifelse(runif(10000)>=0.5,1,0),
  ability = rnorm(10000),
  discrimination = female,
  occupation = 1 + 2*ability + 0*female - 2*discrimination + rnorm(10000),
  wage = 1 - 1*discrimination + 1*occupation + 2*ability + rnorm(10000)

3. 用 DAG 思考一些常見的統計偏誤

3-1. 樣本選擇與衝突因子偏誤

  • 在一些有關電影明星的調查中，主演電影變形金剛的 Megan Fox 被評為 2009 年最差和最具吸引力的女演員 (Piazza 2009)。

  • 這意味著才華和美貌可能是負相關的。為什麼會這樣呢？

  • 才華與美貌在現實中彼此獨立，但控制了「電影明星」（衝突因子）的樣本選擇，導致才華與美貌呈現負相關。

  • 底下的模擬顯示這個現象。

library(tidyverse)
library(patchwork)
set.seed(3444)

star_is_born <- tibble(
  beauty = rnorm(2500),
  talent = rnorm(2500),
  score = beauty + talent,
  c85 = quantile(score, .85),
  star = ifelse(score>=c85,1,0)
)

p1 = star_is_born %>% 
  ggplot(aes(x = talent, y = beauty)) +
  geom_point(size = 1, alpha = 0.5) + xlim(-4, 4) + ylim(-4, 4) + 
  geom_smooth(method = 'lm', se = FALSE) + 
  labs(title = "Everyone")

p2 = star_is_born %>% 
  ggplot(aes(x = talent, y = beauty, color = factor(star))) +
  geom_point(size = 1, alpha = 0.25) + xlim(-4, 4) + ylim(-4, 4) + 
  geom_smooth(method = 'lm', se = FALSE) + 
  labs(title = "Everyone, but different") +
  scale_color_discrete(name = 'Star')

p1 + p2

3-2. 辛普森悖論 (Simpson’s Paradox)

  • WikiPedia 的介紹

  • 簡言之: 數據的趨勢（或型態）會出現在一組資料中，但匯總的趨勢（或型態）可能會逆轉。

  • 一種數學上表示:
    設 \(a_1, b_1, c_1, d_1, a_2, b_2, c_2, d_2>0, \frac{a_1}{b_1}<\frac{c_1}{d_1}, \frac{a_2}{b_2}<\frac{c_2}{d_2},\) 但 \(\frac{a_1+a_2}{b_1+b_2}>\frac{c_1+c_2}{d_1+d_2}.\)

    • 此處，下標 \(1\) 與 \(2\) 表示為兩個類別，或者不同次實驗。
      例如: 類別1表示男生，類別2表示女生

    • 每次類別 \(i\) 或每次實驗 \(i\) 皆分為實驗組與對照組，\(d_i\) 是實驗組的總受測者樣本數，\(c_i\) 是實驗組中（接受新療法後）好轉的樣本數; \(b_i\) 是對照組的總受測樣本數，\(a_i\) 是對照組中（接受傳統療法後）好轉的樣本數。

    • 在兩次實驗中，實驗組表現皆好於對照組，但加總後，對照組表現好於實驗組。

  • 圖解如下圖:

  • 紅箭頭表示實驗組，藍箭頭表示對照組，\(x\) 軸為各自組別的受測樣本數，\(y\) 軸為各自組別的好轉樣本數。

  • 紅箭頭1 比 藍箭頭1 更傾斜，並且 紅箭頭2 比 藍箭頭2 更傾斜，表示兩次實驗的實驗組表現皆好於對照組。

  • 匯總後，藍箭頭+ 反而比 紅箭頭+ 更傾斜: 對照組好於實驗組。

    • 若 藍箭頭2 比 藍箭頭1 與 紅箭頭1 都還要傾斜，那麼，藍箭頭2 夠長就會發生逆轉現象。

4. 小結

  • 沒有資料集帶有「衝突因子」和「混雜因子」的標籤。

  • 估計策略是否符合後門標準的唯一方法是使用 DAG，而一個 DAG 就是一個模型。

  • 換句話說，如果不做出假設，就無法確定 \(D\) 對 \(Y\) 的因果效應。

  • 建立模型需要了解各變數的資料生成過程，這包含我們選擇排除什麼路徑。排除路徑的唯一方法是藉由領域知識與邏輯。

  • 這是無法避免的──所有的實證工作都需要理論來指導。

  • 採用非理論方法來研究經驗主義只會失敗。（即使是「大數據」也無法解決這個問題。）

  • 若無正確的模型，人們就無法制定可靠的辨識策略。

  • 如果我們希望獲得對因果效應的可信估計，必須認真對待您正在研究的現象背後的行為理論。

  • DAG 是一個有用的工具，可以幫助我們思考並表達這些問題。
